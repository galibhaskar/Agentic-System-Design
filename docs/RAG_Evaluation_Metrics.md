# RAG Evaluation Metrics

### Metrics:
1. Precision@k:
    > Determines the amount of noise retrieved

    Precision@k = Retrieved Relevant Items / k

2. Recall@k:
    > Determines the amount of missing information

    Recall@k =  Retrieved Relevant Items / Total Relevant Items

3. Hit Rate@k:
    > Determines whether any relevant information is retrieved at all

    Hit Rate@k = 1 if at least one relevant chunk is retrieved in top k else 0

4. MRR@k (Mean Reciprocal Rank):
    > Determines the ranking order of first relevant chunk
    
    MRR@k = 1/rank of the first relevant chunk in top k

5. NDCG@k (Normalized Discounted Cumulative Gain):
    > Determines the ranking order of all relevant chunks, giving more weight to higher-ranked chunks

    NDCG@k = (DCG@k) / (IDCG@k)
   
   - DCG@k = sum((graded relevance score at rank i) / log2(i + 1)) for i in range(1, k+1), 
   
   - where ``graded relevance score`` is most useful chunk -> 3, somewhat useful chunk -> 2, not useful chunk -> 0

   - ``log2(i + 1)`` is used to discount higher-ranked chunks more weight, as they are more likely to be seen by the user.  

   - IDCG@k is the ideal DCG@k, which is the DCG@k for a perfect ranking of the chunks


6. Faithfulness:
    > Determines whether LLM answered using the context or hallucinated?

    - Uses LLM as judge:
        - Break the answer into individual claims.
        - for each claim, ask LLM: "Is this claim supported by the retrieved chunks? Answer with 'Yes' or 'No' and explain why."
        - Faithfulness Score = Number of claims supported by retrieved chunks / Total number of claims

7. Answer Relevance:
    > Determines whether the answer is relevant to the question or not.

    - Uses LLM as judge:
        - Ask LLM: "what question does this answer best respond to?" - repeat n times.
        - measure semantic similarity between the original question and the questions generated by LLM to evaluate relevance of the answer.
        - Answer Relevance Score = Average semantic similarity between original question and questions generated by LLM
    
8. Factual Correctness:
    > Determines whether answer matches the ground truth or not.

    - Uses LLM as judge:
        - compare LLM answer with ground truth answer using semantic similarity to evaluate factual correctness of the answer.
        - Factual Correctness Score = Semantic similarity between LLM answer and ground truth answer


``Example:`` If we have 5 chunks in the corpus and the relevant chunks for a query are [chunk1, chunk3], and the retrieved chunks in top 3 are [chunk2, chunk1, chunk4], then:
- **Precision@3** = 1/3 (only chunk1 is relevant in top 3)

- **Recall@3** = 1/2 (only chunk1 is retrieved out of the 2 relevant chunks)

- **Hit Rate@3** = 1 (at least one relevant chunk is retrieved in top 3)
- **MRR@3** = 1/2 (the rank of the first relevant chunk, chunk1, is 2)
- **NDCG@3** = (3/log2(2+1)) / (3/log2(1+1) + 3/log2(2+1)) = 0.63 (only chunk1 is relevant in top 3 and it is ranked 2nd, so it gets a discounted relevance score)
- **Faithfulness Score** = 1 (if the claim in the answer is supported by chunk1)
- **Answer Relevance Score** = 0.8 (if the semantic similarity between original question and questions generated by LLM is 0.8 on average)
- **Factual Correctness Score** = 0.9 (if the semantic similarity between LLM answer and ground truth answer is 0.9)
